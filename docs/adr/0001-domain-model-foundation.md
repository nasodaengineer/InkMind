# ADR-0001: 领域模型地基

**Status**: Accepted

## 语境

InkMind 面向超长篇（百万字级）AI 协作写作。需要确定小说、角色、世界观三大核心领域的基础结构，包括层级粒度、状态流转和敏感度模型。这些决策将影响存储层、Agent 协作协议和前端展示的所有下游设计。

## 决策

### 小说层级

采用四层结构（三层强制 + 一层可选）：

```
Novel (强制)
  └── Volume (可选)   — 仅超长篇 (>30万字) 或系列作品使用
       └── Part (强制)     — 情节大段落，如三幕结构
            └── Chapter (强制) — 最小写作单元，1000-3000 字
```

- Chapter 是 AI 生成的最小原子单位，一次生成一章。
- Chapter 具有四态状态机：`大纲 → 写作中 → 待评审 → 已定稿`。
- 每章保存完整的历史版本（ChapterVersion），支持回退和 Diff。

### 来源追踪

段落级（Paragraph）追踪内容来源和校验和，而非章节级。原因：同一章可能混合 AI 生成和人工修订内容，段落级追踪能精确标识哪些段落已被用户修改，防止后续 Agent 协作时误覆盖用户定稿。

### 角色模型

- 角色标记角色类型（主角/反派/配角/龙套），用于指导 AI 戏份分配。
- 性格通过 `性格标签[]` + `行为规则(自由文本)` 表达，而非大五人格量化。更适合叙事场景，AI 更容易保持稳定。
- 别称列表是必填字段，用于指代消歧。
- 角色关系纯文字描述，无量化亲密度。
- 提供独立的 `CharacterTimelineEntry` 实体，记录角色在每章的关键事件，跨章节一致性维护用。
- 无阶段性状态机 — 角色状态通过 `current_state` 自由文本承载即可。

### 世界观模型

- 世界观包含 `MagicSystem` 作为独立结构化实体，而非塞入通用 `rules` 文本。
- 提供结构化 `Location` 层级树（大陆→区域→城市→建筑），支持地点间的父子关系和导航。

## 被否决的方案

- **大五人格 trait**: 心理学量化模型对叙事场景精度过高且不稳定，AI 难以稳定区分 7 分和 8 分的差异。改用标签+规则更务实。
- **扁平 Novel → Chapter**: 对百万字长篇导航效率太低，四层结构是必要的层级深度。
- **纯二元章状态**: 无法支撑写作工作流中的协作评审，四态状态机是 Agent 协作的最低要求。
- **角色阶段性状态机**: 过度设计。角色的阶段性变化（如健康→受伤→濒死）天然可以被 `current_state` 自由文本 + `CharacterTimelineEntry` 覆盖，增加一个专门的状态机只会提升系统复杂度而不增加表达力。
